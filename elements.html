웹 제작 과정

Static Web Development Notes

인스타 피드에 올리기 애매한건 여기 모아둠ㅎ

프로젝트 개요

본 웹페이지는 GitHub 기반 아카이브로, 사진 | 기록 | 일상 로그를 인스타 피드가 아닌 개인 웹에 저장하기 위해 제작했다.

CMS/Framework(React, Vue)를 사용하지 않고, HTML / CSS / JavaScript로 구현한 Static Web

Advantages

  • 사용자 통제 (UI, 콘텐츠 구조, 배포 방식)
  • 반응형 웹 (PC / Tablet / Mobile)
  • Git-hub를 통한 서버 운영 비용 Zero 구조
  • 예측 가능한 웹 페이지 설계·유지

Code Structure [1]
최신 게시물 자동 상단 정렬

본 Web의 경우 Static이라 DB가 존재하지 않는다.
따라서 업로딩 및 수정 순서 → 최신이라는 문제를 DOM 조작으로 해결

안전한 방식

  • 타일을 통째로 뒤집는 방식 채택

HTML

<!-- index.html -->
							  <section class="tiles" id="tiles">
								<!-- article들이 여기에 나열됨 -->
							  </section>

JavaScript

document.addEventListener('DOMContentLoaded', () => {
								const tiles = document.getElementById('tiles');
								const items = Array.from(tiles.children);
							  
								// 최신 게시물이 위로 오도록 역순 재배치
								items.reverse().forEach(el => tiles.appendChild(el));
							  });

Main Point

  • 작성자가 코드 추가한 순서 → 게시 순서
  • 날짜 파싱이나 메타데이터 관리 불필요
  • 로직 단순, 디버깅 쉬움

또한, 본 구현에서는 DOM 접근 전 Guard Logic을 포함하여 페이지 구조 변경 시에도 Run Time Error가 발생하지 않도록 설계했다. 


								  document.addEventListener('DOMContentLoaded', () => {
									const tiles = document.getElementById('tiles');
									if (!tiles) return;
								  
									const items = Array.from(tiles.children);
									items.reverse().forEach(el => tiles.appendChild(el));
								  });

Code Structure [2]
스크롤 고정 미디어 플레이어(UI 지속성 확보)

게시물 열람도중 다른 HTML 페이지로 이동하더라도 음악 재생이 중단되지 않도록 상단 고정 미디어 플레이어를 도입

사용자는 게시물을 스크롤하거나 다른 섹션으로 이동해도 동일한 위치에서 음악 제어가능

position: fixed를 사용하여 스크롤과 무관하게 플레이어 위치를 고정, 메뉴 레이어와의 충돌을 방지하기 위해

z-index를 지정하였다.


Main Point

  • 스크롤 중 음악 제어 가능
  • 콘텐츠 끊김 방지
  • Static 웹에서 미디어 UX 구현

Code Structure [3]
반응형 Grid 기반 피드 레이아웃 설계

본 Web의 메인 화면은 이미지 중심의 게시물을 노출시키기 위해 카드(Tile) 기반 피드 레이아웃으로 설계하였다.

단순히 화면을 축소·확대하는 반응형이 아닌, 디바이스 해상도에 따라 열(Column) 개수가 자연스럽게 변경되는 구조를 채택 하였다.

CSS (Grid 기반 레이아웃)

.tiles {
								display: grid;
								grid-template-columns: repeat(3, 1fr);
								gap: 1.2rem;
							  }
							  
							  @media screen and (min-width: 980px) {
								.tiles {
								  grid-template-columns: repeat(5, 1fr);
								}
							  }

Grid를 사용함으로써 각 타일의 크기와 간격을 일관되게 유지할 수 있었으며, 디바이스별 분기 처리도 직관적으로 관리할 수 있었다.


Main Point

  • 모바일 환경에서도 피드 밀도를 유지하기 위해 2열 채택
  • Tablet / Desktop에서는 점진적으로 열 개수 증가
  • JavaScript 없이 CSS만으로 레이아웃 제어

Lists

Core Design

  • Static web archive built with HTML / CSS / JavaScript
  • Mobile-first responsive layout via CSS media queries
  • GitHub Pages hosting [HTTPS, no server maintenance]
  • Content-first structure: tiles → detail pages → reusable components
  • DOM-based feed ordering to simulate “latest first” without DB
  • External integrations (LinkedIn/Instagram/Google Form/GitHub links)

Page Build

  1. Define content structure (HTML)
  2. Establish responsive layout (CSS)
  3. Add minimal interaction (JavaScript)
  4. Validate mobile-first behavior
  5. Deploy via GitHub Pages
